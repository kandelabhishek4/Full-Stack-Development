CET138 Web Development

What is Full Stack Development?

Full Stack Development covers the frontend, backend, databases, APIs, testing, and deployment needed to create a complete web application.

FS

Overview

Full Stack Development covers both the frontend (client-side) and backend (server-side) of a web application. A Full Stack Developer understands every layer — interface design, server logic, databases, and APIs — needed to build a complete, working product. Rather than specialising in just one part of an application, a full stack developer can move between writing the visual interface a user sees, the server logic that processes requests, and the database queries that store and retrieve information, giving them a broad view of how all the pieces fit together. This makes them especially valuable on smaller teams or projects, where being able to troubleshoot and contribute across the entire system saves time and reduces the need to hand work between specialists. Building genuine full stack skill takes time, since it means becoming comfortable with multiple languages, tools, and ways of thinking, but it ultimately leads to a much deeper understanding of how modern web applications are actually built and deployed.

Frontend

The frontend is what users interact with directly in their browser.

  • HTML — structures the page content, giving every heading, paragraph, and image a defined place and meaning within the document.
  • CSS — handles styling and visual presentation, controlling colours, spacing, fonts, and how the layout responds to different screen sizes.
  • JavaScript — adds interactivity and dynamic behaviour, allowing the page to respond to clicks, update content, and communicate with the backend without reloading.

Backend

The backend handles server-side logic, processing requests and returning responses.

  • Data processing, authentication, and business logic, which run on the server rather than the user's browser, keeping sensitive operations and rules out of public view.
  • Common languages: Node.js, Python, PHP, Java, C#, each offering frameworks that make it faster to build and maintain server-side applications.
  • Example: verifying login credentials and returning success or failure, a typical backend task that checks submitted data against stored records before responding to the frontend.

Databases

Databases store, manage, and retrieve application data.

  • Relational — MySQL, PostgreSQL; structured tables of rows and columns, well suited to data with clear relationships, such as users, orders, and products.
  • Non-relational — MongoDB; flexible, document-style data, useful when the structure of the data may vary or change frequently between records.

APIs & Workflow

APIs bridge the frontend and backend, typically exchanging data in JSON via REST.

  • User interacts with the frontend, such as clicking a button or submitting a form, which triggers the next step in the workflow.
  • Frontend sends a request to the backend through an API, packaging up the relevant data so the server knows what action is being requested.
  • Backend processes the request and queries the database if needed, applying any business logic before deciding what data to return.
  • Response returns to the frontend, which updates the UI, so the user sees the result of their action without needing to reload the page.

Deployment

The final stage hosts the application so users can access it online.

  • Cloud platforms: AWS, Vercel, Netlify, Heroku, which host the application's files and code so it can be reached over the internet rather than only running locally.
  • Performance optimisation and security checks, carried out before and after launch to keep the site fast and protect it from common vulnerabilities.
  • Scalability planning for growing traffic, ensuring the application can keep working smoothly as more users start to access it.
Combining frontend, backend, databases, APIs, and deployment makes Full Stack Developers highly versatile across the entire software development lifecycle.
