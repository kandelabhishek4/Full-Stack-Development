HTML

HTML creates the structure, meaning, and accessible foundation of a web page.

HTML

Overview

HTML (HyperText Markup Language) is the standard markup language used to structure content on the web. It defines the meaning of elements such as headings, paragraphs, images, links, tables, and forms, working alongside CSS for styling and JavaScript for interactivity. Every website, no matter how visually complex, begins as an HTML document, since this is the layer that tells the browser what each piece of content actually is rather than just how it should appear. Learning HTML well means understanding not just how to write tags, but how to choose the right element for the right purpose, which makes pages easier to maintain, more accessible to assistive technology, and better understood by search engines. Because it forms the foundation that CSS styles and JavaScript brings to life, a solid grasp of HTML is the natural starting point for anyone learning web development.

Document Structure

Every HTML document follows the same basic structure.

  • <!DOCTYPE html> — tells the browser this is HTML5, so the page renders using modern standards mode rather than an older, less consistent rendering mode.
  • <head> — metadata, title, character encoding, linked files; nothing here is shown directly on the page, but it controls the tab title, search engine description, and which CSS or JavaScript files are connected.
  • <body> — the visible content shown to users, including all text, images, forms, and other elements that make up the actual page.

Elements & Attributes

Content is organised using elements and tags, most with an opening and closing tag.

  • Example: <p>Paragraph</p> — the opening tag starts the element and the closing tag ends it, with the content sitting in between.
  • href attribute sets a link's destination, telling the browser which page, file, or section to open when the link is clicked.
  • src attribute sets an image's source, pointing the browser to the file location so it knows what to display.

Semantic HTML

HTML5 introduced semantic elements that clearly describe the purpose of content.

  • <header>, <nav>, <main> — mark out the page's top banner, navigation links, and primary content area, so each part of the layout has a clear, recognised role.
  • <article>, <section>, <aside>, <footer> — group self-contained content, thematic sections, side content, and closing information, instead of relying on generic <div> tags.
  • Improves readability, SEO, and accessibility, since search engines and screen readers can understand the structure of the page rather than just a flat list of elements.

Forms & Accessibility

HTML provides built-in tools for collecting input and supporting all users.

  • Text, email, password fields, radio buttons, checkboxes, dropdowns — each input type is suited to a different kind of data, which keeps forms intuitive for users to fill in correctly.
  • Validation attributes: required, minlength, type="email" — these let the browser check user input automatically, catching obvious mistakes before the form is even submitted.
  • Descriptive alt text, labels, and ARIA attributes for screen readers, ensuring that users with visual or motor impairments can still understand and navigate the form.
HTML also supports multimedia via <audio> and <video> — a strong grasp of HTML is the essential first step toward becoming a Full Stack Developer.
